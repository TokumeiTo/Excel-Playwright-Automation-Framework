ðŸ”¹ðŸ”¹ðŸ”¹Why should use PLAYWRIGHT over SELENIUM for this projectðŸ”¹ðŸ”¹ðŸ”¹
1. Network Capabilities 
	- Playwright natively intercepts and listens to all network requests and responses.
	- Selenium does not natively capture network traffic. To capture network requests/responses, we usually need:
	BrowserMob Proxy (a separate proxy server)
	Or Chrome DevTools Protocol (CDP) integration via Selenium (more complex)
Both options add extra setup and can be fragile, especially if multiple browsers are involved.
             ?Why Playwright is Preferable for Network Logging?
	-Directly supported, works cross-browser (Chromium, Firefox, WebKit).
	-Integrates seamlessly with our per-test logging in Excel.
	-No external proxies or tools are required.
	-Easy to track request-response timing, which is essential for reporting performance or detecting failures.

2. Integration with Node.js & Excel
     Node.js-based, using XLSX + ExcelJS.
Playwright has first-class support in Node.js (async/await, modern JS). Selenium is JavaScript-compatible, but its Node.js ecosystem is older and more verbose.
Advantage: Playwright â†’ cleaner, modern async handling.

3. Speed & Reliability
Playwright is faster due to direct browser protocol control.
Selenium uses WebDriver API, which is slightly slower and prone to timing issues.
Advantage: Playwright â†’ more reliable for headless execution and CI/CD pipelines.

4. Headless & Headed Modes
Playwright: Switching between headless and headed is just a parameter. Works seamlessly across browsers.
Selenium: Headless mode is supported, but some browsers (like Safari/WebKit) require more configuration.
 Advantage: Playwright â†’ smoother multi-browser automation.

5. Browser Support & Versions
   Playwright: Bundles browsers internally (Chromium, Firefox, WebKit). We donâ€™t need to install or manage browser versions separately. This makes our test setup more consistent across machines.
   Selenium: Relies on installed browsers + corresponding drivers (chromedriver, geckodriver). We have to keep them in sync; mismatched versions break tests.
Advantage: Playwright â†’ easier setup, fewer version conflicts.
